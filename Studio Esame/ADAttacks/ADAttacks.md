## Attacchi
- <span style="color:rgb(50, 50, 255)">ARP Spoofing:</span>
	- Invio di pacchetti ARP falsificati per associare il proprio indirizzo MAC a un IP legittimo, intercettando o manipolando il traffico.
	- **Obiettivo:**<span style="color:rgb(255, 0, 0)"> Intercettazione del traffico (Man-In-The-Middle), dirottamento a DoS.</span>
	- <b>Mitigazione:</b><span style="color:rgb(0, 255, 0)"> Dynamic ARP Inspection (DAI), stati ARP entries.</span>
- <span style="color:rgb(50, 50, 255)">DHCP Spoofing:</span>
	- Creazione di un server DHCP falso per distribuire configurazioni IP malevole ai dispositivi.
	- **Obiettivo:** <span style="color:rgb(255, 0, 0)">Controllo del traffico di rete, intercettazione dei dati.</span>
	- <b>Mitigazione:</b><span style="color:rgb(0, 255, 0)"> DHCP Snooping, autenticazione delle porte switch.</span>
- <span style="color:rgb(50, 50, 255)">DHCP Starvation:</span>
	- Invio di numerose richieste DHCP per esaurire le risorse del server DHCP legittimo.
	- **Obiettivo:** <span style="color:rgb(255, 0, 0)">Creare un DoS contro il server DHCP.</span>
	- <b>Mitigazione:</b><span style="color:rgb(0, 255, 0)"> Limiti di rate sulle richieste DHCP, DHCP Snooping.</span> 
- <span style="color:rgb(50, 50, 255)">MAC Flooding:</span>
	- Invio di una grande quantità di indirizzi MAC falsi a uno switch per saturarne la tabella CAM.
	- **Obiettivo:** <span style="color:rgb(255, 0, 0)">Fare sì che lo switch inizi a inoltrare il traffico a tutte le porte, favorendo sniffing di rete.</span>
	- <b>Mitigazione:</b><span style="color:rgb(0, 255, 0)"> Port Security, limitazione degli indirizzi MAC per porta.</span> 
- <span style="color:rgb(50, 50, 255)">Spanning Tree Protocol (STP) Attack:</span>
	- Invio di una grande quantità di indirizzi MAC falsi a uno switch per saturarne la tabella CAM.
	- **Obiettivo:** <span style="color:rgb(255, 0, 0)">Fare sì che lo switch inizi a inoltrare il traffico a tutte le porte, favorendo sniffing di rete.</span>
	- <b>Mitigazione:</b><span style="color:rgb(0, 255, 0)"> Port Security, limitazione degli indirizzi MAC per porta.</span> 
- <span style="color:rgb(50, 50, 255)">VLAN Hopping:</span>
	- Forzare il traffico a uscire dalla VLAN assegnata
		- Metodo 1: Double Tagging (aggiunta di due tag VLAN)
		- Metodo 2: Switch Spoofing (simulazione di un trunk)
	- **Obiettivo:** <span style="color:rgb(255, 0, 0)">Intercettazione del traffico tra VLAN diverse</span>
	- <b>Mitigazione:</b><span style="color:rgb(0, 255, 0)"> Disabilitare il trunking non necessario, utilizzare VLAN Access Control List (VACL)</span> 
- <span style="color:rgb(50, 50, 255)">Man-In-The-Middle (MITM) a livello L2:</span>
	- Utilizzo di tecniche come ARP Spoofing o DHCP Spoofing per posizionarsi tra due dispositivi in comunicazione.
	- **Obiettivo:** <span style="color:rgb(255, 0, 0)">Intercettare, modificare o bloccare il traffico</span>
	- <b>Mitigazione:</b><span style="color:rgb(0, 255, 0)"> Autenticazione 802.1X, crittografia (TLS/IPSec)</span>
